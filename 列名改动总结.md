## 🔑 关键技术点

### 1. SQL中的列名处理

连字符是SQL中的减号符号，需要用反引号包围：

```sql
-- ✅ 正确用法
SELECT `open-id`, `nick-name`, `create-time` FROM t_user;

-- ❌ 错误用法（会被解析为减法运算）
SELECT open-id, nick-name, create-time FROM t_user;
```

### 2. MyBatis列名映射

在resultMap中指定列名到属性的映射：

```xml
<resultMap id="UserResultMap" type="User">
    <id column="id" property="id" />
    <result column="open-id" property="openId" />
    <result column="nick-name" property="nickName" />
    <result column="create-time" property="createTime" />
</resultMap>
```

### 3. INSERT/UPDATE语句

```sql
-- ✅ 正确：列名用反引号
INSERT INTO t_user (`open-id`, `nick-name`, `create-time`)
VALUES (?, ?, ?);

-- ✅ 在MyBatis中对应：
<insert id="insert" parameterType="User">
    INSERT INTO t_user (`open-id`, `nick-name`, `create-time`)
    VALUES (#{openId}, #{nickName}, #{createTime})
</insert>
```

## 📋 MyBatis XML Mapper 编写规范

所有Mapper XML文件都应该遵循此模式：

```xml
<!-- 结果映射 -->
<resultMap id="xxxResultMap" type="Xxx">
    <id column="id" property="id" />
    <result column="column-name" property="propertyName" />
    ...
</resultMap>

<!-- 查询 -->
<select id="selectXxx" resultMap="xxxResultMap">
    SELECT `id`, `column-name`, ... FROM t_xxx
    WHERE `id` = #{id}
</select>

<!-- 插入 -->
<insert id="insert" parameterType="Xxx" useGeneratedKeys="true" keyProperty="id">
    INSERT INTO t_xxx (`column1`, `column2`)
    VALUES (#{property1}, #{property2})
</insert>

<!-- 更新 -->
<update id="update" parameterType="Xxx">
    UPDATE t_xxx
    SET `column1` = #{property1}
    WHERE `id` = #{id}
</update>

<!-- 删除 -->
<delete id="delete" parameterType="long">
    DELETE FROM t_xxx WHERE `id` = #{id}
</delete>
```

## 📌 注意事项

### 1. SQL查询时必须使用反引号

```sql
✅ SELECT `open-id` FROM t_user;
❌ SELECT open-id FROM t_user;  -- 错误！会被解析为 open 减去 id
```

### 2. 驼峰式属性自动转换不适用

```properties
# 虽然启用了下划线到驼峰式转换
mybatis.configuration.map-underscore-to-camel-case=true

# 但它只能处理下划线(_)，不能处理连字符(-)
# 因此必须在resultMap中手动配置列名映射
```

### 3. Entity属性保持驼峰式

```java
// ✅ 正确：Java属性使用驼峰式
private String openId;
private String nickName;
private LocalDateTime createTime;

// ❌ 错误：不要改为连字符
private String open-id;  // 这是非法的Java语法
```

### 4. 数据库连接字符编码

确保application.properties中的连接URL包含正确的字符编码：

```properties
spring.datasource.url=jdbc:mysql://localhost:3306/wechat_ordering?useSSL=false&serverTimezone=Asia/Shanghai&characterEncoding=utf8mb4
```
